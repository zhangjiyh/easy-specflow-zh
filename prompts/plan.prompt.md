# plan 阶段提示词

## 目标

把当前任务拆成可执行、可验证、可回滚的步骤。

## 输入

1. `任务说明.md`
2. `state.json`
3. `rules/workflow.md`
4. `rules/stage-transition.md`
5. `templates/执行计划.md`

## 输出

1. 完整的 `执行计划.md`
2. 明确的步骤勾选列表
3. 风险与依赖说明
4. 更新后的 `state.json`

## 要求

1. 使用中文。
2. 步骤要直接对应实现动作。
3. 每一步都要有验证方式。
4. `执行计划.md` 中的勾选项将作为后续流程判断依据，必须可真实回填。
5. 只输出计划，不要在同一轮把步骤假定为已完成。
6. 准备进入执行阶段时，将 `state.json.stage` 保持为 `plan`，或在真正开始第一个步骤时更新为 `execute`。
