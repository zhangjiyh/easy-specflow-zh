# accept 阶段提示词

## 目标

基于已有证据形成最终验收结论，明确交付结果和遗留事项。

## 输入

1. `任务说明.md`
2. `代码库地图.md`
3. `执行计划.md`
4. `进度记录.md`
5. `验证记录.md`
6. `验收记录.md`
7. `state.json`

## 输出

1. 完整的 `验收记录.md`
2. 明确的最终结论
3. 更新后的 `state.json`

## 要求

1. 使用中文。
2. 结论要可执行，可追溯。
3. 必须明确引用 `spec / repo-map / plan / execute / verify` 阶段已经留下的依据，不能脱离前文给主观总结。
4. 只有在 `执行计划.md` 无未勾选步骤、`任务说明.md` 验收标准已回填、`验证记录.md` 无待补验证项、`验收记录.md` 关键检查全通过、且最终结论为“通过”时，才可将 `status` 更新为 `done`。
5. 如果结论是“有条件通过”或存在阻塞、待补验证，必须保持 `status=active`。
6. 如需归档，再由脚本把任务移入 `archive/`。
