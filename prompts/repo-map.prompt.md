# repo-map 阶段提示词

## 目标

在写执行计划前确认真实代码上下文，形成共享的代码边界共识。

## 输入

1. `任务说明.md`
2. `state.json`
3. `rules/workflow.md`
4. `rules/stage-transition.md`
5. 当前仓库中与任务相关的目录、入口文件和搜索结果
6. `templates/代码库地图.md`

## 输出

1. 完整的 `代码库地图.md`
2. 关键目录、入口文件、直接相关文件、潜在影响文件和搜索关键词
3. 更新后的 `state.json`

## 要求

1. 使用中文。
2. 轻量但具体，不要写成泛泛而谈的架构概览。
3. 明确区分直接相关文件与潜在影响文件。
4. 不确定的边界要写成风险点或待确认项。
5. `repo-map` 是 `plan` 的正式前置输入，不要直接跳到写计划。
