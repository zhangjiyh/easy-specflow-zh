# execute 阶段提示词

## 目标

按计划连续执行当前任务的剩余步骤，直到全部已承诺步骤完成或遇到阻塞，并把过程沉淀到共享文档中。

## 输入

1. `任务说明.md`
2. `代码库地图.md`
3. `执行计划.md`
4. `进度记录.md`
5. `state.json`

## 输出

1. 完成本轮连续执行所需的代码或文档改动
2. 在 `进度记录.md` 中记录每个已完成步骤的执行摘要
3. 勾选 `执行计划.md` 中已完成的步骤
4. 更新 `state.json.lastAction`

## 要求

1. 默认连续推进剩余已承诺步骤，直到全部完成或遇到阻塞。
2. 记录每个步骤对应的步骤编号、实际改动、影响文件、验证命令和验证结果。
3. 每完成一个步骤后，必须立即勾选 `执行计划.md` 中对应项。
4. 默认不要因为单步完成就中断整轮执行；对用户在本轮结束后统一总结。
5. 发现阻塞时先写入 `进度记录.md`，并保持 `status=active`。
6. 只要仍有未勾选步骤，就不要进入 `verify` 或 `accept`。
7. 当前阶段保持为 `execute`，直到所有已承诺步骤真实完成。
