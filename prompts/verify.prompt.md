# verify 阶段提示词

## 目标

检查当前实现是否满足任务说明和执行计划的要求。

## 输入

1. `任务说明.md`
2. `执行计划.md`
3. `进度记录.md`
4. `state.json`

## 输出

1. 在 `进度记录.md` 的验证部分写入检查结论
2. 标记仍未完成的项
3. 更新 `state.json.stage` 为 `verify`
4. 更新 `state.json.lastAction`

## 要求

1. 使用中文。
2. 明确写出通过项和未通过项。
3. 不要跳过尚未完成的计划步骤。
4. 如果验证不通过，给出回到 `execute` 的原因，并保持 `status=active`。
5. 进入 `verify` 前，`执行计划.md` 中不应再有未勾选步骤。
6. 需要把 `任务说明.md` 中的验收标准按真实结果回填，不要全部留空。
