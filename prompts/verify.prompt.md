# verify 阶段提示词

## 目标

检查当前实现是否满足任务说明和执行计划的要求，并形成结构化验证证据。

## 输入

1. `任务说明.md`
2. `代码库地图.md`
3. `执行计划.md`
4. `进度记录.md`
5. `验证记录.md`
6. `state.json`

## 输出

1. 在 `验证记录.md` 中写入结构化验证结论
2. 标记仍未完成的项与待补验证项
3. 更新 `state.json.stage` 为 `verify`
4. 更新 `state.json.lastAction`

## 要求

1. 使用中文。
2. 明确写出通过项和未通过项。
3. 不要跳过尚未完成的计划步骤。
4. 如果验证不通过，给出回到 `execute` 的原因，并保持 `status=active`。
5. 进入 `verify` 前，`执行计划.md` 中不应再有未勾选步骤。
6. 验证记录至少应包含验证范围、命令、预期结果、实际结果、是否通过、失败摘要和遗留风险。
7. 需要把 `任务说明.md` 中的验收标准按真实结果回填，不要全部留空。
