# 命名规范

## 任务 ID 规范

任务 ID 固定格式：

```text
TYYYYMMDD-序号
```

示例：

```text
T20260329-001
T20260329-002
```

说明：

1. 日期部分使用任务创建当天。
2. 序号固定为三位数。
3. 同一天内序号递增。

## 目录命名规范

任务目录固定格式：

```text
TYYYYMMDD-序号-任务标题
```

示例：

```text
T20260329-001-登录与鉴权重构
```

要求：

1. 保留中文标题。
2. 去掉不适合作为路径的字符。
3. 避免生成额外 slug 目录。

## 中文文档命名规范

任务目录中的默认文档名固定为：

1. `任务说明.md`
2. `执行计划.md`
3. `进度记录.md`
4. `验收记录.md`

## 状态文件命名规范

状态文件固定为：

```text
state.json
```

任务索引文件固定为：

```text
.specflow/index.json
```

## 命名一致性要求

1. 五阶段命名统一使用 `spec / plan / execute / verify / accept`。
2. 文档名统一使用中文，不为不同平台再造别名。
3. 规则、模板、脚本和文档中的路径说明必须统一使用 `.specflow/`。
