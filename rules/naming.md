# 命名规范

## 任务 ID 规范

任务 ID 固定格式：

```text
TYYYYMMDD-序号
```

示例：

```text
T20260329-001
T20260329-002
```

说明：

1. 日期部分使用任务创建当天。
2. 序号固定为三位数。
3. 同一天内序号递增。

## 目录命名规范

任务目录固定格式：

```text
TYYYYMMDD-序号-任务标题
```

示例：

```text
T20260329-001-登录与鉴权重构
```

要求：

1. 保留中文标题。
2. 去掉不适合作为路径的字符。
3. 避免生成额外 slug 目录。

## 中文文档命名规范

任务目录中的默认文档名固定为：

1. `任务说明.md`
2. `代码库地图.md`
3. `执行计划.md`
4. `进度记录.md`
5. `验证记录.md`
6. `验收记录.md`

## 状态文件命名规范

状态文件固定为：

```text
state.json
```

任务索引文件固定为：

```text
.specflow/index.json
```

## 六阶段中英文名称对照

| 英文名 | 中文名 | 核心产物 |
|--------|--------|----------|
| `spec` | 需求定义 | `任务说明.md` |
| `repo-map` | 代码库定位 | `代码库地图.md` |
| `plan` | 执行计划 | `执行计划.md` |
| `execute` | 执行实现 | `进度记录.md` |
| `verify` | 验证汇总 | `验证记录.md` |
| `accept` | 验收结论 | `验收记录.md` |

说明：
1. 阶段内部标识和脚本命令使用英文名（如 `specflow.sh stage plan`）。
2. 文档名称和对外说明优先使用中文名。
3. 模板中的"文档所属阶段"字段保持英文值，便于脚本解析。

## 命名一致性要求

1. 六阶段命名统一使用 `spec / repo-map / plan / execute / verify / accept`。
2. 文档名统一使用中文，不为不同平台再造别名。
3. 规则、模板、脚本和文档中的路径说明必须统一使用 `.specflow/`。
