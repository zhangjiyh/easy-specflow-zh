## 3. 结构化步骤说明

### Step 1：重构后端登录接口与 JWT 鉴权拦截器

- 步骤编号：1
- 步骤目标：让后端登录接口统一返回 token，并让受保护接口按 JWT 校验。
- 修改范围 / 涉及文件：`AuthController.java`、`AuthService.java`、`JwtTokenInterceptor.java`、`SecurityConfig.java`
- 执行动作：调整登录接口返回结构，接入 JWT 校验逻辑，补白名单配置。
- 验证方式：运行后端测试并使用 Postman 验证登录成功与未授权访问。
- 完成标准：后端登录成功能签发 token，受保护接口未携带 token 时返回 401。
- 当前状态：已完成

### Step 2：接入前端登录页、用户 store 和请求拦截器

- 步骤编号：2
- 步骤目标：让前端能够保存 token，并在请求中自动注入。
- 修改范围 / 涉及文件：`index.vue`、`user.ts`、`http.ts`、`router/index.ts`
- 执行动作：登录成功后保存 token，拦截器注入头部，401 时跳回登录页。
- 验证方式：手动联调登录成功、接口访问和 token 失效跳转。
- 完成标准：前端登录流程完整，未授权跳转可用。
- 当前状态：已完成

### Step 3：完成联调、文档同步和验收准备

- 步骤编号：3
- 步骤目标：同步说明文档并形成验证证据。
- 修改范围 / 涉及文件：联调说明、`验证记录.md`、`验收记录.md`
- 执行动作：汇总接口联调结果，记录失败摘要和待补验证项。
- 验证方式：执行 `./mvnw test`、`pnpm build`，并手动复核关键链路。
- 完成标准：验证证据足以支撑 `accept`，并明确是否存在待补验证项。
