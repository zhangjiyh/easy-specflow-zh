# Vue + Spring 示例

这个目录提供一个面向真实研发场景的使用示例，帮助你把 `easy-specflow-zh` 落到前后端联调任务中。

## 典型任务

任务标题：

`登录与鉴权重构`

## 推荐拆法

### `spec`

在 `任务说明.md` 中写清：

1. 登录成功后返回什么。
2. 前端保存什么状态。
3. 后端如何校验受保护接口。
4. 哪些内容本次不改。

### `plan`

在 `执行计划.md` 中可以拆成：

1. 后端登录接口调整。
2. 后端鉴权拦截器整理。
3. 前端登录流程接入。
4. 前端请求拦截器补齐。
5. 联调与回归。

### `execute`

执行阶段建议同步使用 `templates/checklist-vue-spring.md` 检查：

1. 页面交互。
2. 接口契约。
3. Spring 分层实现。
4. 工作流文档是否补齐。

### `verify`

重点检查：

1. 计划中的步骤是否都已勾选。
2. `进度记录.md` 是否留下验证结论。
3. 登录成功、失效 token、未授权访问等核心场景是否都被验证。

### `accept`

在 `验收记录.md` 中写明：

1. 最终交付内容。
2. 已覆盖的验证场景。
3. 仍需后续跟进的问题。

## 建议命令

```bash
bash scripts/specflow.sh new "登录与鉴权重构"
bash scripts/next-step.sh
bash scripts/verify.sh
```
