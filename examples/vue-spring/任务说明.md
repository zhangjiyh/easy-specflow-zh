## 2. 背景

- 业务背景：前端登录成功后仅缓存用户信息，后端受保护接口仍依赖旧 session 鉴权。
- 现状问题：前后端登录态不统一，token 失效、未授权跳转和接口联调容易出问题。
- 触发原因：团队计划统一为 JWT 鉴权，并同步前后端拦截器行为。

## 3. 目标

- 目标 1：后端登录接口返回 access token，受保护接口按 JWT 校验。
- 目标 2：前端登录页、store 和请求拦截器统一接入 token 流程。
- 目标 3：联调时能覆盖登录成功、token 失效、未授权访问三个核心场景。

## 4. 范围

### 本次范围

- 后端登录接口、鉴权拦截器和安全配置
- 前端登录页、用户 store、请求拦截器
- 联调说明和验证记录

### 非目标

- 不改动注册、找回密码和权限后台管理页面
- 不引入新的 SSO 平台

## 5. 约束与边界

- 禁止无关重构：不重写整体权限模型，只统一登录与接口鉴权链路。
- 禁止未授权依赖升级：保持现有 Vue、Spring Boot 和 JWT 库版本不变。
- 禁止擅自修改部署 / 基础设施：不改网关、不改 Nginx。
- 其他限制：要兼容现有前端路由守卫，不影响其他页面已有权限判断。
